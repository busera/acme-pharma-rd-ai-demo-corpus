# FDA computer software assurance for production and quality management system software

FDA's computer software assurance guidance describes a risk-based approach for establishing confidence in software automation used in production or quality management systems. It focuses on intended use, risk, appropriate rigor, and testing activities.

## Audit-planning relevance

- The direct scope is production and quality management system software, not every R&D AI tool.
- The risk-based assurance concept is still useful when ACME Pharma classifies AI use cases by intended use, data impact, record impact, and quality/regulatory relevance.
- Audit should avoid treating all AI pilots as requiring the same validation package; the assurance approach should scale with risk and intended use.
